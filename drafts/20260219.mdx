❯ fetch 말고 다른 방법 API 호출하는방법이 뭐가 있어?

⏺ API 호출 방법들

  ┌───────────┬──────────────────────────┬───────────────────────────┬────────────────────┐
  │           │          fetch           │           axios           │         ky         │
  ├───────────┼──────────────────────────┼───────────────────────────┼────────────────────┤
  │ 설치      │ 없음 (내장)              │ npm i axios               │ npm i ky           │
  ├───────────┼──────────────────────────┼───────────────────────────┼────────────────────┤
  │ 응답 처리 │ res.json() 수동 호출     │ 자동 JSON 파싱            │ 자동 JSON 파싱     │
  ├───────────┼──────────────────────────┼───────────────────────────┼────────────────────┤
  │ 에러 처리 │ 4xx/5xx에서 에러 안 던짐 │ 자동으로 에러 던짐        │ 자동으로 에러 던짐 │
  ├───────────┼──────────────────────────┼───────────────────────────┼────────────────────┤
  │ 인터셉터  │ 없음                     │ 있음 (요청/응답 가로채기) │ 훅으로 제공        │
  ├───────────┼──────────────────────────┼───────────────────────────┼────────────────────┤
  │ 크기      │ 0kb                      │ ~13kb                     │ ~3kb               │
  └───────────┴──────────────────────────┴───────────────────────────┴────────────────────┘

  ---
  fetch의 불편한 점

  ```javascript
  // fetch — 매번 해줘야 하는 것들이 많음
  const res = await fetch('/api/products');
  if (!res.ok) throw new Error('서버 에러');  // 수동 에러 체크
  const data = await res.json();               // 수동 JSON 파싱
  ```

  axios로 같은 코드

  ```javascript
  import axios from 'axios';

  // GET — 깔끔
  const { data } = await axios.get('/api/products');

  // POST
  const { data } = await axios.post('/api/products', { title: '메모' });

  // DELETE
  await axios.delete(`/api/products/${id}`);
  ```

  res.json() 안 해도 되고, 4xx/5xx면 자동으로 에러가 던져짐.

  ---
  axios가 프로젝트에서 많이 쓰이는 이유: 인터셉터

  ```javascript
  // 모든 요청에 토큰 자동 첨부
  axios.interceptors.request.use((config) => {
    config.headers.Authorization = `Bearer ${getToken()}`;
    return config;
  });

  // 모든 응답에서 401이면 로그아웃 처리
  axios.interceptors.response.use(
    (res) => res,
    (error) => {
      if (error.response?.status === 401) {
        logout();
      }
      return Promise.reject(error);
    }
  );
  ```

  인증, 에러 처리, 로깅을 한 곳에서 관리할 수 있습니다.

  ---
  실무에서는 axios가 가장 많이 쓰이고, 가벼운 프로젝트에서는 ky나 그냥 fetch도 충분합니다. 지금 프로젝트에서는 fetch로 계속 진행해도 괜찮아요. 나중에 인터셉터가 필요한
  순간이 오면 axios로 바꾸면 됩니다.
